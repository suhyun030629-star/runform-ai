# RunForm AI

**RunForm AI**는 러닝 영상을 입력하면 MediaPipe Pose 기반 관절 좌표를 추정하고, 달리기 자세를 이해하기 쉬운 문장으로 피드백하는 웹 애플리케이션입니다.  
결과는 운동 자세를 점검하기 위한 참고용이며 의료적 진단이나 부상 진단을 대신하지 않습니다.

## 1. 프로젝트 목적

러닝 자세를 점검하려면 촬영 영상을 반복해서 보거나 전문가의 도움을 받아야 합니다. RunForm AI는 단일 측면 러닝 영상에서 다음 지표를 자동 산출해, 사용자가 스스로 자세 경향을 확인하도록 돕습니다.

- 상체 기울기와 양쪽 팔꿈치 각도
- 착지 순간의 무릎 굽힘각 및 좌우 차이
- 착지 후보에서의 전족·중족·후족 착지 경향
- 골반 이동 벡터를 기준으로 계산한 오버스트라이드
- 수직 진폭과 케이던스
- 영상 구도에 따른 결과 신뢰도

## 2. 핵심 기술과 분석 흐름

1. **Pose estimation**: MediaPipe Pose가 프레임별 33개 신체 랜드마크를 추정합니다.
2. **특징 추출**: 어깨·골반·팔꿈치·무릎·발목 좌표에서 각도와 상대 위치를 계산합니다.
3. **이동 방향 추정**: 시작·종료 구간의 골반 중앙값 차이로 2D 이동 벡터를 구합니다.
4. **착지/보폭 분석**: 발목 높이의 국소 최대값을 착지 후보로 보고, 발목-골반 벡터를 이동 방향에 투영합니다.
   같은 프레임의 발끝과 뒤꿈치 높이 차이로 전족·중족·후족 경향도 표시합니다.
5. **신뢰도 처리**: 화면상 몸 크기 변화(원근)와 이동 벡터의 수평 성분을 함께 확인합니다. 대각선·정면·왕복 구도에서는 오버스트라이드와 수직 진폭을 `참고용` 또는 `확인 필요`로 표시합니다.
6. **피드백 생성**: 목표 범위와 비교한 뒤, 단순 수치 대신 “어떻게 고칠지”를 한국어 문장으로 제안합니다.

### 분석 항목과 기준

| 항목 | 기본 목표/판정 방식 |
| --- | --- |
| 상체 기울기 | 5–10°를 기본 목표 범위로 사용 |
| 팔꿈치 각도 | 80–100°를 기본 목표 범위로 사용 |
| 착지 무릎 굽힘각 | 15–25°를 기본 목표 범위로 사용 |
| 수직 진폭 | 5–10 cm를 참고 범위로 사용하며 구도 신뢰도가 낮으면 수치 판정을 보류 |
| 케이던스 | 170–180 SPM을 참고 범위로 사용 |
| 오버스트라이드 | 이동 벡터 앞쪽으로의 발목-골반 상대거리를 신장으로 정규화해 계산 |
| 착지 유형 | 발끝·뒤꿈치의 화면상 높이 차이로 전족·중족·후족 경향을 참고용으로 표시 |

> 위 목표 범위는 일반적인 러닝 폼 점검을 위한 휴리스틱입니다. 촬영 각도, 속도, 개인 특성에 따라 달라질 수 있습니다.
>
> 착지 유형은 단일 영상에서 계산한 경향이며, 어떤 유형이 모든 사람에게 더 좋다는 판정이 아닙니다. 통증, 속도, 훈련 목적과 함께 해석해야 합니다.

## 3. 데이터 활용

프로젝트에서 제공된 **사람동작영상 데이터**의 러닝 영상을 점검용으로 활용했습니다.  
`video_action_2/2-1`은 같은 동작을 여러 각도에서 촬영한 영상, `2-2`는 다른 동작을 여러 각도에서 촬영한 영상으로 구성되어 있습니다.

이 구조를 바탕으로 동일 동작의 촬영 각도가 달라질 때 결과가 과도하게 흔들리지 않도록 다음을 반영했습니다.

- 픽셀 거리 대신 **신장 대비 비율**로 정규화
- 왼쪽→오른쪽/오른쪽→왼쪽을 가정하지 않고 **골반 전체 이동 벡터**를 자동 추정
- 사람이 화면에서 커지거나 작아지는 정도를 원근 변화로 산출
- 구도가 불리하면 수치를 단정하지 않고 신뢰도 안내를 표시

## 4. 사용 방법

### 촬영 조건

- 7–15초 길이의 영상
- 카메라는 고정하고, 전신과 발이 모두 보이게 촬영
- 가능한 한 측면에서 한 방향으로 달리기
- 시작 전 1–2초는 카메라 앞에 가만히 서 있기
- 화면에는 분석 대상 **한 사람만** 나오게 하기

### Docker 실행 (권장)

1. Docker Desktop을 설치한 뒤 실행합니다.
2. `running_pose_web` 폴더에서 `RunForm_Docker.command`를 더블클릭합니다.  
   또는 터미널에서 아래 명령을 실행합니다.

   ```bash
   docker compose up --build
   ```

3. 로그에 `Listening at: http://0.0.0.0:8510`이 나타나면 브라우저에서 [http://127.0.0.1:8510](http://127.0.0.1:8510)을 엽니다.
4. 종료하려면 실행 중인 터미널에서 `Ctrl+C`를 누르거나 아래를 실행합니다.

   ```bash
   docker compose down
   ```

Docker 컨테이너는 `/health` 상태 점검 경로를 제공하며, 업로드 및 분석 영상은 호스트의 `web_uploads` 폴더에 보관됩니다.

### 로컬 실행

Python 3.10 환경에서 다음을 실행합니다.

```bash
pip install -r requirements.txt
python flask_app.py
```

또는 `RunForm_AI.command`를 더블클릭합니다.

## 5. 프로젝트 구조

```text
running_pose_web/
├── flask_app.py          # 웹 화면, 업로드, 분석 요청, 결과 렌더링
├── running_analyzer.py   # Pose 추정, 지표 계산, 피드백 생성
├── app.py                # 단일 WSGI 진입점
├── Dockerfile            # 컨테이너 이미지 정의
├── docker-compose.yml    # 포트·볼륨·상태 점검 설정
├── requirements.txt      # 재현 가능한 패키지 버전
├── RunForm_Docker.command# macOS Docker 실행 도우미
└── web_uploads/          # 업로드/분석 결과 영상 (Docker 볼륨)
```

## 6. 배포 및 재현성

- **공개 서비스**: [RunForm AI (Hugging Face Spaces)](https://isis0629-runform-ai.hf.space/)
- **배포 소스**: [Hugging Face Space 저장소](https://huggingface.co/spaces/isis0629/runform-ai)
- Flask + Gunicorn으로 애플리케이션을 실행합니다.
- Hugging Face Spaces의 무료 Docker 환경에서 `flask_app.py`와 `running_analyzer.py`를 실행합니다.
- 배포 환경은 7860 포트에서 서비스하며, 로컬 Docker 환경은 `docker-compose.yml`을 통해 8510 포트에서 동일한 앱을 실행할 수 있습니다.
- 업로드한 개인 영상과 분석 결과 영상은 공개 저장소에 포함하지 않습니다.
- `Dockerfile`이 Python, 시스템 라이브러리, 분석 패키지를 고정된 순서로 설치합니다.

## 7. 한계와 향후 개선

- 단일 RGB 영상 기반이므로 정밀한 3D 관절각·실제 거리 측정에는 한계가 있습니다.
- 대각선, 정면, 왕복, 카메라 이동 영상에서는 거리 기반 지표의 신뢰도가 떨어질 수 있습니다.
- 향후에는 다중 카메라 또는 깊이 정보, 더 정교한 착지 이벤트 모델, 개인별 속도·부상 이력 기반 기준을 추가할 수 있습니다.
